# Caching

Manifest discovery involves a recursive filesystem scan that can touch hundreds of directories. The caching layer eliminates this cost on every request after the first: results are persisted to a backend and restored on subsequent boots.

---

## How It Works

On the first boot (**cold boot**):

1. The Locator scans the filesystem and finds every `locator.manifest`.
2. Each manifest is parsed and its namespace, symbols, and virtuals are registered.
3. The discovery results are **serialised and written to the cache backend**.

On subsequent boots (**warm boot**):

1. The Locator reads the cached payload.
2. Namespaces, symbols, and virtuals are restored without touching the filesystem.
3. The full cold-boot scan is skipped entirely.

```
Cold boot:  scan → parse → register → save cache          ~10–200 ms
Warm boot:  load cache → restore                           ~1–5 ms
```

> **Note:** Caching applies only when discovering from the **default root directory** (either `$_SERVER["DOCUMENT_ROOT"]` in web contexts or `getcwd()` in CLI). Scans that target a custom directory via `discoverManifests($customDir)` are never cached automatically.

## Backends

### File-Based (default)

When Wingman Stasis is **not** installed, the `CacheManager` class writes a PHP-exportable file to disk.

**Default location:** `temp/cache.php` in the Locator package root.

**Override the path:**

```php
$locator = new Locator(['locator.caching.file' => '/var/cache/locator.php']);
```

**File format:** A PHP file returning a plain array:

```php
<?php
# Generated by Wingman Locator - 2026-03-13 12:00:00
return [
    'manifests' => [ /* dehydrated manifest arrays */ ],
    'roots'     => [ /* scanned root + profile pairs */ ],
    'timestamp' => 1741862400,
];
```

Manifest source paths are stored relative to the package root so the cache file is portable and safe to copy between identical installations.

**Write safety:** The file is written using `LOCK_EX` (`file_put_contents($path, $content, LOCK_EX)`), making concurrent PHP-FPM writes atomic. Two workers racing on first boot will each write the file but only one lock-holder writes at a time, preventing a corrupted partial write.

### Wingman Stasis (when installed)

When [Wingman Stasis](https://github.com/angelpolitis/Wingman-Stasis) is detected at runtime, the `Bridge\Stasis\CacheManager` automatically replaces the file-based manager. It stores the discovery payload as a single key-value entry under the key `locator.discovery.cache`.

**Supported adapters:** Local filesystem, Redis, Memcached, or any adapter compatible with the Wingman Stasis adapter contract.

**Provide a custom adapter:**

```php
use Wingman\Stasis\Adapters\RedisAdapter;
use Wingman\Locator\Locator;

$locator = new Locator(
    ['locator.caching.ttl' => 3600],
    new RedisAdapter('127.0.0.1', 6379)
);
```

## TTL (Time-To-Live)

Set `locator.caching.ttl` to a positive integer (seconds) to automatically invalidate stale caches.

```php
$locator = new Locator(['locator.caching.ttl' => 7200]); // 2-hour TTL
```

- **File backend:** A `timestamp` field is written alongside the data; on load, the age is compared against the configured TTL. Stale entries cause a new cold-boot scan.
- **Cacher backend:** The TTL is passed directly to Cacher's `set()` call; expiry is managed by the adapter (e.g. Redis TTL).
- **`0`** (the default): The cache never expires automatically. Clear it manually or after a deployment.

## Clearing the Cache

**Programmatically:**

```php
use Wingman\Locator\Locator;
use ReflectionObject;

$locator = Locator::get();
$ref = new ReflectionObject($locator);
$prop = $ref->getProperty('cacheManager');
$prop->setAccessible(true);
$prop->getValue($locator)?->clear();
```

**Via CLI (recommended):**

```bash
wingman locator:cache:clear
```

See [console.md](console.md) for details.

---

## Disabling Caching

```php
$locator = new Locator(['locator.caching.enabled' => false]);
```

With caching disabled, every request performs a full filesystem scan. Use this during development when manifest files change frequently, or in test environments where a consistent cold-boot is required.

---

## Cache File Housekeeping

Add the cache file (or the `temp/` directory) to `.gitignore` to avoid committing environment-specific paths:

```
# .gitignore
modules/Locator/temp/
```

After a deployment that adds, removes, or renames namespaces, clear the cache:

```bash
wingman locator:cache:clear
```

Or delete the cache file manually:

```bash
rm modules/Locator/temp/cache.php
```
